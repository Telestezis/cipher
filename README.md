## 🔍 Что вообще делает эта программа?

Это **оконное приложение** на Java, которое **шифрует и расшифровывает текст** с помощью **шифра Гронсфельда** (это разновидность шифра Цезаря, но вместо одной буквы сдвига используется **числовой ключ**, например: `123`, и каждая буква сдвигается на цифру из ключа по очереди).

Программа предназначена **только для русского алфавита** (включая букву **Ё** — 33 буквы всего).

---

## 🧱 Основная структура: класс `GronsfeldCipherGUI`

```java
public class GronsfeldCipherGUI extends JFrame
```

- Это **класс**, который называется `GronsfeldCipherGUI`.
- `extends JFrame` значит: «это окно (графический интерфейс)».
- То есть программа создаёт **окошко на экране**, куда можно вводить текст, ключ и нажимать кнопки.

## 🔐 Основная логика: `processText`

```java
private String processText(String text, String key, boolean encrypt)
```

Это **главная функция**, которая **шифрует или расшифровывает** текст.

### Шаги:

1. **Проверка ключа**:

   ```java
   if (!key.matches("\\d+")) {
       JOptionPane.showMessageDialog(... "Ключ должен содержать только цифры!");
       return "";
   }
   ```

   - Ключ должен быть **только из цифр**, иначе выдаст ошибку.

2. **Проходим по каждой букве текста**:

   ```java
   for (char c : text.toCharArray()) {
   ```

3. **Если символ — буква**:

   - Преобразуем в **заглавную**, ищем её **номер в алфавите**.
   - Если буква **не русская** — оставляем как есть (например, латиница, цифры, знаки).

4. **Вычисляем сдвиг**:

   ```java
   int shift = Character.getNumericValue(key.charAt(keyIndex % key.length()));
   ```

   - Берём цифру из ключа (циклически: если ключ `123`, то сдвиги: 1,2,3,1,2,3,1...).
   - `keyIndex % key.length()` — чтобы не выйти за пределы ключа.

5. **Если шифруем**:

   ```java
   newIndex = (charIndex + shift) % 33;
   ```

   - Добавляем сдвиг, но если вышли за конец алфавита — **возвращаемся к началу** (остаток от деления).

6. **Если расшифровываем**:

   ```java
   newIndex = (charIndex - shift + 33) % 33;
   ```

   - Вычитаем сдвиг. `+33` нужно, чтобы **не получить отрицательное число**.

7. **Сохраняем регистр**:

   - Если была **маленькая буква** — делаем результат маленьким.
   - Если **заглавная** — оставляем заглавной.

8. **Если символ не буква** (пробел, знак, цифра):

   - Просто копируем его без изменений.

9. Возвращаем готовую строку.

## 🖱 Обработчики кнопок

### `EncryptAction` — когда нажимаешь «Зашифровать»

```java
String text = inputArea.getText();
String key = keyField.getText().trim();
if (key.isEmpty()) { ... предупреждение ... }
String result = processText(text, key, true);  // true = шифровать
outputArea.setText(result);
```

### `DecryptAction` — аналогично, но `processText(..., false)` — **расшифровать**.

Если ключ не введён — покажет предупреждение.

## 🧠 Пример работы

**Ввод:**

```
Привет!
Ключ: 314
```

**Шифрование:**

- П → (номер 16) + 3 = 19 → С
- р → (номер 17) + 1 = 18 → с (маленькая!)
- и → (8) + 4 = 12 → м
- в → (2) + 3 = 5 → е
- е → (5) + 1 = 6 → ж
- т → (19) + 4 = 23 → ч

**Результат:** `Ссмежч!`

(Символ `!` остаётся без изменений.)

---

## 💡 Важные моменты

- **Только русские буквы шифруются**. Латиница, цифры, знаки — не трогаются.
- **Регистр сохраняется**: если ввели «ПрИвЕт», результат будет с теми же заглавными буквами.
- **Ключ должен быть цифрами**: `123` — ок, `abc` или `12a` — ошибка.
- Если ключ короче текста — цифры **повторяются по кругу**.
